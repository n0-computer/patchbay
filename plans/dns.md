## Phase 1 — Per-device `/etc/hosts` overlay via Rust API

**Goal**: `Lab::dns_entry()` and `Device::dns_entry()` let callers register
name→IP mappings. Spawned commands (`dev.spawn_command(cmd)`) see the entries
in `/etc/hosts` via a per-process mount-namespace bind-mount. In-process Rust
code uses `Lab::resolve(name)` for the same lookups.

### Rust API

```rust
use std::net::IpAddr;

/// Record types for dns_entry.
pub enum DnsRecord {
    A(Ipv4Addr),
    AAAA(Ipv6Addr),
}

impl Lab {
    /// Adds a hosts entry visible to all devices.
    pub fn dns_entry(&self, name: &str, ip: IpAddr) -> Result<()>;

    /// Resolves a name from the lab-wide + device-specific entries.
    /// For in-process Rust code that can't see bind-mounted /etc/hosts.
    pub fn resolve(&self, name: &str) -> Option<IpAddr>;
}

impl Device {
    /// Adds a hosts entry visible only to this device.
    pub fn dns_entry(&self, name: &str, ip: IpAddr) -> Result<()>;

    /// Resolves a name using this device's entries + lab-wide entries.
    pub fn resolve(&self, name: &str) -> Option<IpAddr>;
}
```

Multiple calls with the same name accumulate addresses (valid for
multi-homed hosts). `dns_entry` can be called before or after `build`.

### How it works

Each device gets a generated hosts file at
`<workdir>/hosts/<ns_name>.hosts`. The file is (re)written on each
`dns_entry` call with the accumulated entries for that device (its own +
lab-wide).

When `spawn_command` forks a child, a `pre_exec` hook:
1. `unshare(CLONE_NEWNS)` — private mount namespace for this process only.
2. `mount --bind <workdir>/hosts/<ns>.hosts /etc/hosts` — overlay the
   generated file over the system hosts file.

This is safe in the rootless/userns setup: `unshare(CLONE_NEWNS)` +
bind-mount within the new mount namespace works without `CAP_SYS_ADMIN`
when inside a user namespace.

For in-process Rust code (`dev.run_sync`, `dev.spawn`), the bind-mount
doesn't apply (threads share the process mount namespace). Use
`dev.resolve(name)` / `lab.resolve(name)` instead — these look up the
in-memory entry map directly.

### Storage

Add to `NetworkCore`:

```rust
struct DnsEntries {
    /// Lab-wide entries: name → Vec<IpAddr>
    global: HashMap<String, Vec<IpAddr>>,
    /// Per-device entries: device NodeId → name → Vec<IpAddr>
    per_device: HashMap<NodeId, HashMap<String, Vec<IpAddr>>>,
    /// Working directory for generated hosts files
    workdir: PathBuf,
}
```

### Hosts file generation

On each `dns_entry` call, regenerate the file for affected device(s):

```
# generated by netsim
127.0.0.1  localhost
::1        localhost

# lab-wide
10.0.1.2   relay
203.0.113.5  server.example.com

# device-specific
10.0.2.3   peer
```

Lab-wide entries go to all devices. Device-specific entries only to that
device.

### pre_exec injection

Modify `Device::spawn_command`, `Router::spawn_command`, `Ix::spawn_command`
to inject the `pre_exec` hook when a hosts file exists for the device:

```rust
pub fn spawn_command(&self, mut cmd: Command) -> Result<std::process::Child> {
    let (ns, netns, hosts_path) = {
        let inner = self.lab.lock().unwrap();
        let dev = inner.device(self.id).ok_or_else(|| anyhow!("unknown device"))?;
        let path = inner.dns.hosts_path_for(self.id);
        (dev.ns.clone(), Arc::clone(&inner.netns), path)
    };
    if let Some(hosts) = hosts_path {
        unsafe {
            cmd.pre_exec(move || {
                if libc::unshare(libc::CLONE_NEWNS) != 0 {
                    return Err(std::io::Error::last_os_error());
                }
                let src = std::ffi::CString::new(hosts.as_os_str().as_bytes())?;
                let dst = c"/etc/hosts";
                if libc::mount(
                    src.as_ptr(), dst.as_ptr(),
                    std::ptr::null(), libc::MS_BIND | libc::MS_RDONLY,
                    std::ptr::null(),
                ) != 0 {
                    return Err(std::io::Error::last_os_error());
                }
                Ok(())
            });
        }
    }
    netns.run_closure_in(&ns, move || {
        cmd.spawn().context("spawn command in namespace")
    })
}
```

## Future work

These phases build on phase 1 but are out of scope for now.

### In-lab authoritative DNS server (hickory-server)

Run a real DNS server inside the lab root namespace so devices can resolve
names via standard DNS queries. Builds on phase 1 by calling
`set_nameserver_all` to point `/etc/resolv.conf` at the in-lab server.
Exposes `DnsServer` handle with `set_a`, `add_aaaa`, `add_txt`, `remove`,
`records` methods and runtime mutation.

### In-lab ACME CA (pebble) + cert trust injection

Run pebble as a local ACME CA for TLS certificate provisioning in tests.
Inject pebble's root CA into spawned processes via environment variables
(`SSL_CERT_FILE`, etc.) or bind-mount over the system CA bundle. Includes
DNS-01 challenge responder that auto-satisfies challenges via the in-lab
DNS server.
